# SPEC.md

## Intent vs Implementation
- **Goal:** Provide developers with ready-to-use code examples for Gemini API.
- **Status:** Functional for learning, insufficient for production readiness.
- **Mapping:** 
    - The repository successfully acts as a cookbook.
    - Implementation currently prioritizes "ease of copy-pasting" over robust software engineering patterns (error handling, security, rate limiting).
- **Living Spec:** The repository will continue to serve as a hub, but should introduce "Production Ready" tags/folders for examples that include security and error handling patterns.
